# ExoTwin

The personal AI "digital twin" experience for individual employees. ExoTwin creates a living representation of a worker's skills, credentials, career progress, and growth trajectory — helping them understand where they are, where they can go, and how to get there.

---

## What it does

- **Identity Card**: the worker's digital profile — name, current role, active badges (Learning, Coaching, etc.), and their AI twin connection status
- **Skills Passport**: a portable, verifiable record of all the employee's skills with proficiency levels and evidence of achievement
- **Career Path Builder**: visual tool for mapping possible career trajectories; shows branching paths from the current role
- **Coaching Feed**: personalized coaching insights, nudges, and recommendations generated by the ExoTwin AI
- **Private Playground**: a personal sandbox where employees can experiment with AI tools and explore ideas without impacting their team
- **Portable Credentials Card**: exportable credential summary for use outside the platform (job applications, professional networks)
- **Collaboration Visualization**: shows how the employee's work intersects and connects with their team's activities
- **Twin Confidence Meter**: a readiness/confidence score showing how well-calibrated the digital twin's model of the employee is
- **Activity Timeline**: chronological feed of the employee's learning events, achievements, and twin interactions

## Architecture & data flow

The ExoTwin components are **presentational** — they receive data via props or render from seeded mock data. They do not manage complex state or call API routes directly in the current prototype.

```
app/dashboard/portal/page.tsx (or integrated panel)
    │
    └─► ExoTwin components (rendered in the portal or as standalone views)
            │
            └─► All data is currently seeded mock data (hardcoded in component files)
```

---

## API integrations

- No direct API route calls in the current prototype
- Future: the Coaching Feed and Career Path Builder will integrate with an AI recommendations API

---

## Data

- All employee data shown (name, skills, badges, timeline events) is **hardcoded mock data** within each component
- No Supabase reads/writes in the current state

---

## Current status & known gaps

- All components are in prototype/showcase state — data is hardcoded
- No real user identity integration (the profile shows "Alex Rivera" as a placeholder)
- No data persistence or actual twin model training
- The Coaching Feed does not yet call an LLM for real-time coaching suggestions

---

## How to extend

- **Connect to real employee data**: wire the Identity Card and Skills Passport to the employee data in the Talent Studio's Supabase tables
- **Add AI coaching**: create an `/api/exotwin-coaching` route that uses the employee's skill profile and recent activity to generate personalized coaching nudges
- **Enable credential export**: implement a PDF or JSON export function in `PortableCredentialsCard.tsx` using the employee's Supabase skill records
- **Build the twin model**: define what "twin confidence" means quantitatively (e.g., % of skills verified, recency of activity) and compute the score dynamically
